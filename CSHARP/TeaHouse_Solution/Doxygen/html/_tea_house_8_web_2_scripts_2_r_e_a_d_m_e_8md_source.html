24 ## Wut? Poppers?
25 
26 A popper is an element on the screen which "pops out" from the natural flow of your application.
27 Common examples of poppers are tooltips, popovers and drop-downs.
28 
29 
30 ## So, yet another tooltip library?
31 
32 Well, basically, **no**.
33 Popper.js is a **positioning engine**, its purpose is to calculate the position of an element
34 to make it possible to position it near a given reference element.
35 
36 The engine is completely modular and most of its features are implemented as **modifiers**
37 (similar to middlewares or plugins).
38 The whole code base is written in ES2015 and its features are automatically tested on real browsers thanks to [SauceLabs](https://saucelabs.com/) and [TravisCI](https://travis-ci.org/).
39 
40 Popper.js has zero dependencies. No jQuery, no LoDash, nothing.
41 It's used by big companies like [Twitter in Bootstrap v4](https://getbootstrap.com/), [Microsoft in WebClipper](https://github.com/OneNoteDev/WebClipper) and [Atlassian in AtlasKit](https://aui-cdn.atlassian.com/atlaskit/registry/).
42 
43 ### Popper.js
44 
45 This is the engine, the library that computes and, optionally, applies the styles to
46 the poppers.
47 
48 Some of the key points are:
49 
50 - Position elements keeping them in their original DOM context (doesn't mess with your DOM!);
51 - Allows to export the computed informations to integrate with React and other view libraries;
52 - Supports Shadow DOM elements;
53 - Completely customizable thanks to the modifiers based structure;
54 
55 Visit our [project page](https://fezvrasta.github.io/popper.js) to see a lot of examples of what you can do with Popper.js!
56 
57 Find [the documentation here](/docs/_includes/popper-documentation.md).
58 
59 
60 ### Tooltip.js
61 
62 Since lots of users just need a simple way to integrate powerful tooltips in their projects,
63 we created **Tooltip.js**.
64 It's a small library that makes it easy to automatically create tooltips using as engine Popper.js.
65 Its API is almost identical to the famous tooltip system of Bootstrap, in this way it will be
66 easy to integrate it in your projects.
67 The tooltips generated by Tooltip.js are accessible thanks to the `aria` tags.
68 
69 Find [the documentation here](/docs/_includes/tooltip-documentation.md).

97 ### Dist targets
98 
99 Popper.js is currently shipped with 3 targets in mind: UMD, ESM and ESNext.
100 
101 - UMD - Universal Module Definition: AMD, RequireJS and globals;
102 - ESM - ES Modules: For webpack/Rollup or browser supporting the spec;
103 - ESNext: Available in `dist/`, can be used with webpack and `babel-preset-env`;
104 
105 Make sure to use the right one for your needs. If you want to import it with a `<script>` tag, use UMD.

107 ## Usage
108 
109 Given an existing popper DOM node, ask Popper.js to position it near its button
110 
111 ```js
112 var reference = document.querySelector('.my-button');
113 var popper = document.querySelector('.my-popper');
114 var anotherPopper = new Popper(
115  reference,
116  popper,
117  {
118  // popper options here
119  }
120 );
121 ```
122 
123 ### Callbacks
124 
125 Popper.js supports two kinds of callbacks, the `onCreate` callback is called after
126 the popper has been initialized. The `onUpdate` one is called on any subsequent update.
127 
128 ```js
129 const reference = document.querySelector('.my-button');
130 const popper = document.querySelector('.my-popper');
131 new Popper(reference, popper, {
132  onCreate: (data) => {
133  // data is an object containing all the informations computed
134  // by Popper.js and used to style the popper and its arrow
135  // The complete description is available in Popper.js documentation
136  },
137  onUpdate: (data) => {
138  // same as `onCreate` but called on subsequent updates
139  }
140 });
141 ```

143 ### Writing your own modifiers
144 
145 Popper.js is based on a "plugin-like" architecture, most of its features are fully encapsulated "modifiers".
146 A modifier is a function that is called each time Popper.js needs to compute the position of the popper. For this reason, modifiers should be very performant to avoid bottlenecks.
147 
148 To learn how to create a modifier, [read the modifiers documentation](docs/_includes/popper-documentation.md#modifiers--object)

151 ### React, Vue.js, Angular, AngularJS, Ember.js (etc...) integration
152 
153 Integrating 3rd party libraries in React or other libraries can be a pain because
154 they usually alter the DOM and drive the libraries crazy.
155 Popper.js limits all its DOM modifications inside the `applyStyle` modifier,
156 you can simply disable it and manually apply the popper coordinates using
157 your library of choice.
158 
159 For a comprehensive list of libraries that let you use Popper.js into existing
160 frameworks, visit the [MENTIONS](/MENTIONS.md) page.
161 
162 Alternatively, you may even override your own `applyStyles` with your custom one and
163 integrate Popper.js by yourself!
164 
165 ```js
166 function applyReactStyle(data) {
167  // export data in your framework and use its content to apply the style to your popper
168 };
169 
170 const reference = document.querySelector('.my-button');
171 const popper = document.querySelector('.my-popper');
172 new Popper(reference, popper, {
173  modifiers: {
174  applyStyle: { enabled: false },
175  applyReactStyle: {
176  enabled: true,
177  fn: applyReactStyle,
178  order: 800,
179  },
180  },
181 });
182 
183 ```

194 ### Performances
195 
196 Popper.js is very performant. It usually takes 0.5ms to compute a popper's position (on an iMac with 3.5G GHz Intel Core i5).
197 This means that it will not cause any [jank](https://www.chromium.org/developers/how-tos/trace-event-profiling-tool/anatomy-of-jank), leading to a smooth user experience.
